BSDGlobalsKFS_Database (Module)

This module contains items that aid with database usage.

Methods

Database.AddConnectionKFS As Boolean

Increments the connection count of the database, tries to connect it if necessary, and returns whether or not the database is connected.

DisconnectAllDatabases

Disconnects each database that is tracked by the Add/ReleaseConnectionKFS functions regardless of connection counts, except for in-memory REALSQLDatabases. Also clears the tracking pool.

Database.HasTableKFS( String ) As Boolean

Returns whether or not the given table exists in the database, using the TableSchema property to avoid potential SQL incompatibilities with different databases.

Database.ReleaseConnectionKFS( [Double] )

Decrements the connection count for the database. If it becomes zero, the database is removed from the tracking pool and is disconnected (unless it is an in-memory REALSQLDatabase). The optional parameter specifies a delayed disconnect, in seconds. If AddConnectionKFS is used on the database before a delayed disconnect expires, then the delayed disconnect is automatically canceled.

Notes

Invoking AddConnectionKFS on a database over and over again increments the connection count of the database. ReleaseConnectionKFS must then be invoked on the database an equal number of times in order for the database to be disconnected. This allows multiple processies to use the same database without needing to know whether or not to disconnect it when they're done.
