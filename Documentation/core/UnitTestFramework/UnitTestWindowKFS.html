UnitTestWindowKFS (Inherits from Window)

UnitTestWindowKFS is a window that provides a fully functional UnitTestArbiterKFS object plus an interactive display interface. Just give it some test classes at runtime, and let it do the rest.

Enumerations

DetailsBoxAutoVisibilityOptions

DetailsBoxAutoVisibilityOptions As Integer

Defines the options for when to automatically show or hide the details box. Can be one of:

  • Hidden
  • Automatic
  • Visible

Methods

ProcessTestClasses

ProcessTestClasses( lst() As UnitTestBaseClassKFS )

ProcessTestClasses( ParamArray lst As UnitTestBaseClassKFS )

Brings the window to the front, and instructs the arbiter to start working on the given test classes.

If you do do not want the window to be brought to the front, then bypass this method by instructing the arbiter directly:

UnitTestWindowKFS.Arbiter.CreateJobsForTestClasses( New TestFoo, New TestBar, New TestFish ... )

Properties

Arbiter

Arbiter As UnitTestArbiterKFS

Provides access to the UnitTestArbiterKFS object inside this window.

AutoSelectErrors

AutoSelectErrors As Boolean

If True, then any time a new test result is displayed, and the test failed, it is selected. Depending on the value of the DetailsBoxAutoVisibility property, this may cause the details box to become visible and display the error.

DetailsBoxAutoVisibility

DetailsBoxAutoVisibility As UnitTestWindowKFS.DetailsBoxAutoVisibilityOptions

Tells the current setting for whether or not to automatically hide or show the details box based on what is selected in the Listbox. Setting this value can affect the DetailsBoxVisible property.

DetailsBoxVisible

DetailsBoxVisible As Boolean

Tells whether or the the details box is currently visible. Setting this value does not affect the value of the DetailsBoxAutoVisibility property.

Compatibility

This class is compatible with console applications, and should work on all platforms.

This class is thread safe.

Notes

Assuming you have test classes named TestFish, TestDog, and TestCat, the following code will make the UnitTestWindowKFS window run and display the results of your test cases. Some common properties were set to provide a examples, but all of those settings were already default.

// Make the Listbox automatically select rows that represent errors (this is default):

UnitTestWindowKFS.AutoSelectErrors = True


// Make the details box appear only when an error is selected in the Listbox (this is default):

UnitTestWindowKFS.DetailsBoxAutoVisibility = UnitTestWindowKFS.DetailsBoxAutoVisibilityOptions.Automatic


// Show the window, and instruct its arbiter to run the test cases:

UnitTestWindowKFS.ProcessTestClasses New TestFish, New TestDog, New TestCat
