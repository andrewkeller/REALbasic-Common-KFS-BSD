Unit Testing (Framework)

The Unit Testing Framework is a collection of classes that allow you to easily write and execute many, small, modular pieces of code that test something.

This page describes how to use the Unit Testing Framework, not why you should use Test Driven Development, although the advantages of Test Driven Development are scattered throughout this document.

Classes

UnitTestArbiterKFS

Class UnitTestArbiterKFS

Runs your test cases, and assembles results into a simple String.

The Unit Testing Framework is implemented using Introspection. You tell a UnitTestArbiterKFS object a list of test classes, and it will identify and invoke the test cases automatically.

UnitTestBaseClassKFS

Class UnitTestBaseClassKFS

Provides a set of methods that greatly increase the ease of writing test cases. A set of test cases is contained within a class, and that class must be a subclass of UnitTestBaseClassKFS.

UnitTestExceptionKFS

Class UnitTestExceptionKFS

Raised when an assertion fails or when you don't catch an exception in a test case.

UnitTestResultKFS

Class UnitTestResultKFS

Contains the results of a single test case. The UnitTestArbiterKFS class has a pool of these when it has completed running your test cases, one for each test method.

UnitTestViewKFS

Class UnitTestResultKFS

A ContainerControl that contains a UnitTestArbiterKFS object, a Listbox, and a StaticText control, and is configured to display the results of unit tests interactively. It's plug-and-play, but it currently cannot display the results of alternate UnitTestArbiterKFS objects.

Compatibility

This framework is compatible with console applications, and should work on all platforms.

This framework is thread safe. The UnitTestArbiterKFS class is guaranteed to never run more than one test case at a time in a given test class, even in a multithreaded environment.
