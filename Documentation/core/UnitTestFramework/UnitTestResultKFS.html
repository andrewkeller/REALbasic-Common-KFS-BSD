UnitTestResultKFS (Inherits from Object)

The UnitTestResultKFS class is used by the UnitTestArbiterKFS class to represent the results of a single test case. It contains all the exceptions that were thrown, and how much time it took the test to run.

Constructors

UnitTestBaseClassKFS, Introspection.MethodInfo, Boolean = True

New UnitTestBaseClassKFS( subject As UnitTestBaseClassKFS, topic As Introspection.MethodInfo, doRunTheTest As Boolean = True )

Creates a new UnitTestResultKFS object that runs, and then represents, the results for the given unit test.

The Boolean parameter is how a skipped test is created. If something basic about the test class fails, then you might not want to run the rest of the test cases. In that case, however, you still want to have result containers for the skipped tests. Solution? A test can be skipped.

Properties

e_ClassSetup()

e_ClassSetup() As UnitTestExceptionKFS

An array of the exceptions raised during the execution of the test class' constructor.

e_Core()

e_Core() As UnitTestExceptionKFS

An array of the exceptions raised during the execution of the test case.

e_Setup()

e_Setup() As UnitTestExceptionKFS

An array of the exceptions raised during the execution of the test case setup event.

e_TearDown()

e_TearDown() As UnitTestExceptionKFS

An array of the exceptions raised during the execution of the test case tear down event.

TestCaseFailed

TestCaseFailed As Boolean

Tells whether or not this test case is considered failed, based on whether or not there are any exceptions. A failed test class constructor currently causes the test case to appear as failed.

TestCasePassed

TestCasePassed As Boolean

Tells whether or not this test case is considered passed, based on whether or not there are any exceptions. If a test case did not pass, then do not assume it is a failure; check whether or not the test case was skipped.

TestCaseSkipped

TestCaseSkipped As Boolean

Tells whether or not this test case was skipped.

TestClassName

TestClassName As String

Tells the name of the test class.

TestMethodName

TestMethodName As String

Tells the name of the test case.

t_ClassSetup

t_ClassSetup As DurationKFS

Tells the time it took for the test class constructor to run.

t_Core

t_Core As DurationKFS

Tells the time it took for the core segment of the test case to run.

t_Setup

t_Setup As DurationKFS

Tells the time it took for the setup event to run for this test case.

t_TearDown

t_TearDown As DurationKFS

Tells the time it took for the tear down event to run for this test case.

Compatibility

This class is compatible with console applications, and should work on all platforms.

This class is thread safe.

Notes

Not all of the properties are are provided as read-only, like they perhaps should be. Try to avoid modifying the contents of a UnitTestResultKFS object after it has been created.
