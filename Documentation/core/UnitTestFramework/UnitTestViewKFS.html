UnitTestViewKFS (Inherits from ContainerControl)

The UnitTestViewKFS class is a container control that provides a fully functional UnitTestArbiterKFS object plus an interactive display interface. Just drag it onto a window, give it some test classes at runtime, and let it do the rest.

Enumerations

DetailsBoxAutoVisibilityOptions

DetailsBoxAutoVisibilityOptions As Integer

Defines the options for when to automatically show or hide the details box. Can be FullManual, FullAutomatic, or HideUntilExceptions.

DetailsBoxPositions

DetailsBoxPositions As Integer

Defines where the details box is located relative to the Listbox. Can be Right or Bottom.

Events

TestFinished

TestFinished( testCaseObject As UnitTestResultKFS )

A duplicate of the TestFinished event of the UnitTestArbiterKFS object inside this container. Raised when the container is done with what it needs to do, and now it's your turn.

TestStarting( testClassName As String, testCaseName As String )

A duplicate of the TestStarting event of the UnitTestArbiterKFS object inside this container. Raised when the container is done with what it needs to do, and now it's your turn.

Properties

Arbiter

Arbiter As UnitTestArbiterKFS

Provides access to the UnitTestArbiterKFS object inside this container control.

AutoSelectErrors

AutoSelectErrors As Boolean

If True, then any time a new test result is displayed, and the test failed, it is selected. Depending on the value of the DetailsBoxAutoVisibility property, this may cause the details box to become visible and display the error.

DetailsBoxAutoVisibility

DetailsBoxAutoVisibility As UnitTestViewKFS.DetailsBoxAutoVisibilityOptions

Tells the current setting for whether or not to automatically hide or show the details box based on what is selected in the Listbox

DetailsBoxPosition

DetailsBoxPosition As UnitTestViewKFS.DetailsBoxPositions

Tells the current position of the details box relative to the Listbox.

DetailsBoxVisible

DetailsBoxVisible As Boolean

Tells whether or the the details box is currently visible. Setting this value does not affect the value of the DetailsBoxAutoVisibility property.

HeadingVisible

HeadingVisible As Boolean

Tells whether or not the heading is currently visible.

Compatibility

This class is compatible with console applications, and should work on all platforms.

This class is thread safe.

Notes

Assuming you have a UnitTestViewKFS object named myUnitTestView on a Window, the following code will make the UnitTestViewKFS object do its thing. All properties were used to provide a good example, but some of those settings were already default.

// Make the Listbox automatically select rows that represent errors:

myUnitTestView.AutoSelectErrors = True


// Make the details box appear only when an error is selected in the Listbox:

myUnitTestView.DetailsBoxAutoVisibility = UnitTestViewKFS.DetailsBoxAutoVisibilityOptions.FullAutomatic


// Make the details box appear below the Listbox, rather than on the right:

myUnitTestView.DetailsBoxPosition = UnitTestViewKFS.DetailsBoxPositions.Bottom


// Make the details box hidden:

myUnitTestView.DetailsBoxVisible = False


// Make the heading visible:

myUnitTestView.HeadingVisible = True


// Instruct the arbiter to run our imaginary test cases:

myUnitTestView.Arbiter.ExecuteTests New TestFish, New TestDog, New TestCat
