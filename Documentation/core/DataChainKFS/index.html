DataChainKFS (Inherits from Object)

This class is a hybrid of a stack and a queue.

The data element is a Variant. For simplicity purposes, this class does not interpret what you pass through it. For example, this class uses the NodeKFS class as its internal data structure. If you pass a NodeKFS object through this class, it will be treated as a single value the whole time. It will not be interpreted as a part of the chain.

This class is implemented using a linked list. It is designed to be fast, sacrificing the ability to behave like an array. All functions that do not deal with the head or the tail of the linked list will potentially take a lot of time to run, due to the concept of searching the entire linked list linearly. There is no way to jump right to any given node.

Constructors

[default]

New DataChainKFS

Creates a new empty DataChainKFS object.

Properties

Count

Count As Integer

Returns the number of items currently stored in the chain.

IsEmpty

IsEmpty As Boolean

Returns True if the chain is currently empty, and False otherwise.

Methods

Append

Append( value As Variant )

Appends the given item to the end of the chain.

Clear

Clear()

Blows away the whole chain, leaving nothing inside the object.

HasValue

HasValue( value As Variant ) As Boolean

Returns whether or not the given item exists in the chain.

Be warned: this function has O(n) complexity. Since this class is designed to be fast with large data sets, it is very possible that this class will be used with large data sets. In that case, O(n) time is relatively huge. Do not rely on this function unless you have to.

Peek

Peek() As Variant

Returns the head of the chain without removing it from the chain.

Pop

Pop() As Variant

Returns the head of the chain, and removes it from the chain.

Push

Push( value As Variant )

Prepends the given item to the beginning of the chain.

Compatibility

This framework is compatible with console applications, and should work on all platforms.

This framework is thread safe.
