NodeKFS (Inherits from Object)

This class is designed to be a generic node class, used for creating more sophisticated data structures.

This class looks so bland because adding more features typically makes things less generic.

Notes

When creating any data structure more advanced than a singly-linked list, you must be concerned with accidentally creating a reference loop. Since REALbasic uses reference counting to know when to deallocate objects, it stands to reason that a set of objects, each with a reference to another in a loop, none of the objects will ever have a reference count of zero, and thus, will never be deallocated. To solve this problem, WeakRefs can be used. A WeakRef contains a reference to an object without counting for that object's reference count.

If you are creating a singly-linked list, then use the Right property to link to the next node.

If you are creating a doubly-linked list, then use the Right and Parent properties to link to the next and previous nodes (respectively).

If you are creating a doubly-linked binary tree, then use the Right and Left properties for linking to the children, and the Parent property for linking to the parent.

In other words, the Parent property is special. Since it is implemented using a WeakRef, it does not count as one of the references to that object. Use the Parent property only, and always, when appropriate.
