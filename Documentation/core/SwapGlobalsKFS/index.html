SwapGlobalsKFS (Module)

This module aids in allocating and releasing temporary files.

Methods

AcquireSwapFile

AcquireSwapFile( bTrack As Boolean = True ) As Folderitem

Returns a file with a unique name that doesn't exist in a folder with read/write permissions. The usage count of this file starts out at 1.

ReleaseAllSwapFiles

SwapGlobalsKFS.ReleaseAllSwapFiles()

Releases all tracked temporary files, regardless of their usage counts. Also clears the tracking pool.

ReleaseSwapFile

ReleaseSwapFile( f As FolderItem )

Decrements the usage count of the given file. If the usage count hits zero, the file is deleted and removed from the tracking pool.

RetainSwapFile

RetainSwapFile( f As FolderItem )

Increments the usage count of the given file.

Notes

Retaining the same swap file over and over again increments the usage count of the file. The file must be released an equal number of times (plus one for the initial acquiring) in order for it to be deleted. This allows multiple processes to use the same swap file without needing to know whether or not to delete it when they're done.
