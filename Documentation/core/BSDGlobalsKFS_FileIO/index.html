BSDGlobalsKFS_FileIO (Module)

This module contains items that aid with accessing the filesystem.

Methods

Folderitem.EqualsKFS( Folderitem ) As Boolean

Returns whether or not the given folderitems in question resolve to the same file in the local filesystem.

GetFolderitemKFS( String ) As Folderitem

Generates a Folderitem from the given path (String), and can parse relative paths.

GetStandardAccessErrorMessageKFS( FolderItem, Boolean, Boolean, Boolean, Boolean, Boolean ) As String

Returns a human-readable error message for not being able to access the given Folderitem, given the provided mustExist, mustBeAFile, mustBeAFolder, mustBeReadable, and mustBeWritable properties.

Folderitem.NextSerialNameKFS( String, [String] ) As String

Returns a name, based on what is given, that is unique in the current directory. Optionally, delimator can be provided between the basename and the number. The result is similar to MyFile 1.txt, MyFile 2.txt, MyFile 3.txt, and so on.

Folderitem.NextSerialNumberKFS( String, String, [Boolean, [String, [String, [String, [Integer]]]]] ) As String

Returns the next number what would make the given file or folder name unique in the current directory, given the part of the name before the number (String), the part of the name after the number (String), whether or not the number zero should be blank (Boolean), the delimator before the number (String), the delimator after the number (String), the format string used to generate the number (String), and the number to start counting at (Integer).

Folderitem.ShellPathKFS( [Boolean, [Boolean]] ) As String

Same as Folderitem.ShellPath in RB's framework except this one optionally includes shell spaces (Boolean), and optionally forces unix slashes on all platforms (Boolean).

Notes

Folderitem.ShellPathKFS doesn't work quite right on all platforms. It should probably be rewritten.
