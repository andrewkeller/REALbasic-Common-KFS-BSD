ResourceManagerKFS (Interface)

The ResourceManagerKFS interface can be used when you want an object to be able to return an object corresponding to a given String key. The behavior is simmilar to that of a Dictionary, except that this interface does not allow write access to the object.

Methods

Fetch

Fetch( key As String, ByRef value As Variant, ByRef persistenceCriteria As PersistenceCriteriaKFS ) As Boolean

A method that should set the 'value' and 'persistenceCriteria' parameters to the values corresponding to the given key, and then return True.

If the given key does not have a value associated with it, or does not have a value that is not expired, then the 'value' and 'persistenceCriteria' parameters should be set to Nil and the method should return False.

Get

Get( key As String ) As Variant

A method that should return the value corresponding to the given key.

If the given key does not have a value associated with it, or does not have a value that is not expired, then a ResourceNotFoundInManagerException should be raised.

Lookup

Lookup( key As String, defaultValue As Variant ) As Variant

A method that should return the value corresponding to the given key.

If the given key does not have a value associated with it, or does not have a value that is not expired, then the 'defaultValue' parameter should be returned.

Compatibility

This interface does not impose any restrictions that cause it to be incompatible with any particular type of application (eg Console, Desktop, and/or Web applications).

This interface does not impose any restrictions that cause it to be unsafe in a threadded environment.

This interface is completely decoupled from the rest of this library.
